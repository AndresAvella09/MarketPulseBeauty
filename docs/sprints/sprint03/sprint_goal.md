**Objetivo del tercer Sprint**

Consolidar la arquitectura de datos del proyecto bajo principios DataOps: migrar el almacenamiento a Parquet particionado, implementar versionado con DVC, establecer contratos de calidad de datos entre etapas del pipeline y construir ingesta incremental. Paralelamente, automatizar la validación del proyecto mediante CI con GitHub Actions y pruebas unitarias. Como resultado visible para el negocio, el dashboard debe mostrar los datos desde la nueva capa de almacenamiento, incorporar el análisis de Google Trends e incluir comparación entre productos.
