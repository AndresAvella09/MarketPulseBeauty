## Planificación del sprint inicial
En este documento se definen inicialmente los objetivos de las tareas planificadas para el primer sprint, el despliegue de cada una se definirán en el documento `backlog.md`.

### Diego
1. Estructura del repositorio: objetivo — establecer la arquitectura de carpetas, espacios de trabajo y documentación inicial para arrancar el proyecto.
2. Entorno y dependencias (`requirements.txt`): objetivo — configurar un entorno virtual reproducible y listar dependencias básicas.
3. Selección de productos: objetivo — definir hasta 3 productos prioritarios para el análisis del negocio.

### Daniel
1. Investigación de fuentes de reseñas públicas: objetivo — identificar fuentes accesibles y coste-efectivas para obtener datos de reseñas.
2. Script de extracción inicial: objetivo — implementar un extractor funcional que recoja reseñas para los productos seleccionados.
3. Estructura y documentación de variables: objetivo — definir y documentar el esquema de datos extraídos.

### Paula
1. Conexión a Google Trends (`pytrends`): objetivo — dejar un script listo para consultas ordenadas y repetibles.
2. Selección de palabras clave: objetivo — optimizar y documentar keywords estratégicas para las consultas.
3. Análisis descriptivo de ratings: objetivo — realizar un EDA de ratings para entender distribución y calidad de los datos.
